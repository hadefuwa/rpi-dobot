👁 Vision System

▼ How the Vision System Works

The Vision System uses computer vision algorithms to detect defects and anomalies in captured images. It connects to a USB camera and provides real-time analysis using OpenCV-based detection methods.

🎥 Camera Connection

  • Connect a USB camera to the Raspberry Pi
  • Click "Connect Camera" to initialize the camera feed
  • The system automatically detects the first available USB camera (index 0)
  • Camera resolution is set to 640x480 pixels at 30 FPS
  • Live stream is displayed in MJPEG format for efficient transmission

🔍 Detection Methods

1. Blob Detection

Uses adaptive thresholding and morphological operations to detect small defects (10-5000 pixels). Best for detecting spots, holes, and small surface imperfections.

2. Contour Analysis

Uses Canny edge detection and contour finding to identify larger defects (50-10000 pixels). Filters defects by area and aspect ratio. Best for detecting cracks, scratches, and irregular shapes.

3. Edge Detection

Uses Canny edge detection and Hough line transform to detect linear defects and edge anomalies. Groups nearby lines to form defect regions. Best for detecting scratches, cuts, and edge defects.

4. Combined (Recommended)

Uses all three methods simultaneously and merges nearby detections to eliminate duplicates. Provides the most comprehensive defect detection by combining the strengths of all methods.

⚙️ How It Works

  1. Image Capture: The system captures a frame from the USB camera
  2. Preprocessing: Converts to grayscale and applies Gaussian blur to reduce noise
  3. Detection: Applies the selected detection method(s) to identify potential defects
  4. Filtering: Removes noise and merges nearby detections (if using combined method)
  5. Analysis: Calculates defect properties (position, size, area, confidence)
  6. Visualization: Draws bounding boxes and labels on the analyzed image

📊 Results Interpretation

  • Defects Found: Yes/No indicator if any defects were detected
  • Defect Count: Total number of unique defects found
  • Confidence: Detection confidence score (0-100%) based on defect coverage and count
  • Method: The detection method used for analysis
  • Defect Details: Each defect shows type, position (x, y), size (width × height), and area

💡 Tips for Best Results

  • Ensure good lighting conditions for clear image capture
  • Keep the camera at a consistent distance from the object
  • Use "Combined" method for general-purpose defect detection
  • Use specific methods (Blob/Contour/Edge) for targeted defect types
  • Enable "Auto-Analyze" for continuous monitoring (analyzes every 3 seconds)
  • Use "Capture" to save images for later review or documentation
  • Check the analyzed image to see visual annotations of detected defects

🔧 Technical Details

  • Library: OpenCV (cv2) for image processing
  • Frame Rate: 30 FPS for live stream
  • Resolution: 640×480 pixels (configurable)
  • Stream Format: MJPEG (Motion JPEG) for efficient web streaming
  • Image Format: JPEG with 85-90% quality for analysis
  • Processing: Real-time analysis with threading support
